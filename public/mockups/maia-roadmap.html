Architecture pipeline d'ingestion

Upload PDF → Supabase Storage → Trigger.dev v9 → Orchestrator

Pipeline A · Texte

  • unpdf → pagesText[]
  • Haiku TOC → chapters[]
  • Sonnet par chapter → questions QCM/numeric/short_text
  • INSERT par chapitre (partial success)

Pipeline B · Images

  • pdfjs canvas → images PNG dedup SHA-256
  • Haiku Vision → classification 81 types + confidence
  • Sonnet image-aware + chapter context → questions
  • Threshold dynamique 0,80 / 0,95 selon affinity matière

Le trigger DB trg_job_auto_done coordonne atomiquement la fin des 2 pipelines (BEFORE UPDATE Postgres trigger SECURITY DEFINER).

Sprints à venir

Ordre proposé, scope identifié

  1. M.1 · DOCX extractor

    Prochain · ~1,5 j

    Parser natif .docx (OOXML = ZIP + XML) :

    • jszip unzip + fast-xml-parser sur word/document.xml
    • Split par Heading 1 → pseudo-pages contigus (1, 2, 3...)
    • Images depuis word/media/*, mapping page via walk linéaire des <w:drawing>
    • Activé derrière flag DOCX_UPLOAD_ENABLED (off par défaut)

  2. M.2 · PPTX extractor

    ~1,5 j

    Parser natif .pptx : 1 slide = 1 page (mapping naturel). Parse ppt/slides/slide{N}.xml + images depuis ppt/media/* liées via rels. Flag PPTX_UPLOAD_ENABLED. Refus explicite .doc/.ppt legacy avec message UX.

  3. M.3 · Wiring upload route + bucket + UI

    ~0,5 j

    Étendre validation route /api/courses/upload-url à .docx / .pptx. Nouveau bucket course-documents (cohabite avec course-pdfs). UI dropzone : "PDF, Word ou PowerPoint" + accept attribute.

  4. Fix race condition chapter mapping

    Tech debt · ~0,5 j

    Déplacer extractChapters dans l'orchestrateur (au lieu de dans runTextPipeline). Partage TOC entre A et B → fini la race où pipeline B fallbackait sur course.title au lieu du vrai chapter. Cosmétique sur PDF mono-chapter mais bloquant sur PDF multi-chapitres (géo 3e, histoire 4e).

  5. Sprint 2B · Vue concept unifiée

    ~2 j

    Refonte curation : vue par concept au lieu de par question. Théorie en champs séparés (définition, formules, exemples), questions liées, slider on/off par concept. Source quote read-only. Préparation à la curation "manuelle" du prof + génération récursive depuis un concept.

  6. Plan Maïa · session quotidienne

    ~3 j

    Session 20 min multi-matière auto-générée chaque matin pour chaque élève. Pick-and-choose, non-adaptatif au skip (équilibré). Heatmap mastery par concept côté élève + prof (max 3-5 alertes simultanées élève, encourageant > exhaustif).

  7. PIN auth · 4 caractères

    ~1,5 j

    Double-auth SSO Google/Microsoft + PIN 4 chars (bcrypt cost 12, partagé par user, timezone user). Fallback SSO après 3 échecs PIN. Tous rôles (prof + élève + admin).

  8. Live session · Kahoot mode année

    ~3 j

    Routes /live et /session (déjà scaffolded). Phase silencieuse écran + random-pick élève réponse orale projetée. Pensée comme mode de cours toute l'année, pas révision ponctuelle.

  9. PWA installable

Dette technique à surveiller

P1 · bloquant à terme

Race condition chapter mapping

Sur PDF multi-chapitres, les images peuvent être attribuées à course.title au lieu du vrai chapter si pipeline B démarre avant pipeline A.

P2 · cosmétique

phase = 'validating' vs status = 'done'

Race entre trigger DB et updateJob final. Le Stepper peut afficher état stale, mais status est correct.

P2 · ops

Vercel auto-deploy cassé

PRs #77 et #79 n'ont pas auto-triggered Vercel. Deploy manuel requis. À investiguer.

P3 · sécurité

Rotation PAT Supabase
